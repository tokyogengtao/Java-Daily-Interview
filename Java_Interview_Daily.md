# Java 每日面试题备份 (2026-03-17)

## 1. JVM 内存模型包含哪些区域？
- **程序计数器**：线程私有，记录当前线程执行的字节码行号。
- **虚拟机栈**：线程私有，存储局部变量表、操作数栈、动态链接等。
- **本地方法栈**：为虚拟机使用的 Native 方法服务。
- **堆 (Heap)**：线程共享，存放对象实例，是垃圾回收（GC）的主要区域。
- **方法区/元空间**：线程共享，存储类信息、常量、静态变量等。

## 2. HashMap 的底层实现原理是什么？
- **数据结构**：JDK 1.8 之后采用 **数组 + 链表 + 红黑树**。
- **转换机制**：当链表长度大于 8 且数组容量大于 64 时，链表会转为红黑树，以提高查询效率（从 O(n) 降低到 O(log n)）。

## 3. Java 中强、软、弱、虚引用的区别？
- **强引用 (Strong)**：普通的 `new` 对象，垃圾回收器绝不回收，内存不足时抛出 OOM。
- **软引用 (Soft)**：内存不足时才会被回收，常用于缓存。
- **弱引用 (Weak)**：只要垃圾回收器扫描到，无论内存是否充足都会回收。
- **虚引用 (Phantom)**：形同虚设，主要用于跟踪对象被回收的状态。

## 4. 什么是双亲委派模型？为什么要使用它？
- **定义**：类加载器收到加载请求时，先委派给父类加载器处理，直到顶层的启动类加载器。只有父类无法加载时，子类才尝试加载。
- **作用**：
    1. **安全性**：防止核心 API（如 `java.lang.String`）被随意替换。
    2. **唯一性**：确保同一个类在不同加载器环境中只被加载一次。

## 5. 线程池的核心参数有哪些？
1. `corePoolSize`：核心线程数。
2. `maximumPoolSize`：最大线程数。
3. `keepAliveTime`：非核心线程空闲存活时间。
4. `unit`：存活时间单位。
5. `workQueue`：任务阻塞队列。
6. `threadFactory`：创建线程的工厂。
7. `handler`：拒绝策略（当任务满时的处理方案）。
